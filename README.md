# ARIA - Adaptive Reasoning Intelligence Agent

ARIA - локальная языковая модель на основе LSTM с обучением через обратную связь (RLHF-like), написанная на Rust. Работает на GPU через OpenCL.

> **Важно:** ARIA поддерживает только видеокарты NVIDIA. Работа на AMD, Intel и других GPU не гарантируется.

## Запуск

```bash
# Клонировать репозиторий
git clone https://github.com/lold/ARIA.git
cd ARIA

# Собрать и запустить (debug)
cargo run

# Собрать оптимизированную сборку
cargo build --release
.\target\release\aria.exe
```

При первом запуске ARIA автоматически:
1. Создаёт папку `data base/` с пустыми файлами датасетов
2. Строит словарь из текстовых файлов в `data base/`
3. Проводит предобучение (pre-training) на этих данных
4. Сохраняет модель в `aria_model.json` и токенизатор в `aria_tokenizer.json`

При последующих запусках модель загружается из сохранённых файлов.

## Обучение

### Что нужно для обучения

- Видеокарта **NVIDIA** (обязательно, вычисления идут через OpenCL/CUDA)
- VRAM: рекомендуется от **4 ГБ** (модель: embed=500, hidden=1200)
- RAM: от **8 ГБ**
- Текстовые датасеты в папке `data base/`

### Файлы датасетов

Положи свои тексты в папку `data base/` (создаётся автоматически):

```
data base/
  Books.txt        - художественные книги
  Reddit.txt       - диалоги, посты
  Poetry.txt       - поэзия
  summary.txt      - краткие изложения
  words.txt        - словарный запас
  news.txt         - новостные тексты
```

Чем больше текста, тем лучше качество. Рекомендуемый минимум: **10-50 МБ** на файл.

### Процесс обучения

ARIA обучается в двух режимах:

**1. Предобучение (pre-training)** - происходит автоматически при первом запуске на основе файлов из `data base/`. Модель учится предсказывать следующее слово.

**2. Дообучение через фидбек (RLHF)** - происходит в реальном времени во время диалога. После каждого ответа ARIA спрашивает оценку:

```
Rate (I like / I dont like / skip):
```

- `I like` - положительный сигнал, модель усиливает этот паттерн
- `I dont like` - отрицательный сигнал, логируется для дальнейшего использования
- `skip` (или Enter) - пропустить без обновления

## Команды в диалоге

| Команда | Описание |
|---|---|
| `stats` | Показать статистику сессии |
| `settings` | Показать текущие настройки генерации |
| `mode greedy` | Жадная генерация (детерминированная) |
| `mode topk` | Top-K сэмплинг (k=20, по умолчанию) |
| `mode topp` | Nucleus (top-p) сэмплинг (p=0.9) |
| `temp <0.1-2.0>` | Установить температуру генерации |
| `topk <n>` | Установить значение K |
| `topp <0.0-1.0>` | Установить значение P |
| `exit` | Выйти |

## Возможные проблемы

**`error: linker link.exe not found`**
→ Установи Visual Studio Build Tools с компонентом C++ (см. раздел выше)

**Модель не видит GPU**
→ Убедись, что установлен драйвер NVIDIA и CUDA Toolkit. OpenCL должен быть доступен через драйвер NVIDIA.

**Медленное предобучение**
→ Это нормально при большом датасете. Для ускорения используй `cargo build --release` вместо `cargo run`.
